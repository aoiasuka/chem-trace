# 高校实验室危化品全流程追溯系统 —— 运行说明（Next.js 实现）

## 一、技术栈

- Next.js 16（App Router）+ React 19 + TypeScript
- Prisma 6 + SQLite（开箱即用，无需安装数据库服务）
- Auth.js (NextAuth v5) —— 登录认证 + 路由保护
- Tailwind CSS v4 —— 界面样式
- Zod —— 表单校验

## 二、首次运行步骤

```bash
cd chem-trace

# 1. 安装依赖（若 node_modules 已存在可跳过）
npm install

# 2. 初始化数据库（生成 dev.db 与表结构）
npx prisma migrate dev

# 3. 写入种子数据（管理员账号 + 5 条示例危化品）
npm run seed

# 4. 启动开发服务器
npm run dev
# 访问 http://localhost:3000
```

生产模式：

```bash
npm run build
npm run start
```

## 四、功能与五人模块对应

| 路由 | 模块 | 课题功能 | 负责人 |
|------|------|----------|--------|
| `/chemicals` | 危化品基础信息库 + MSDS 弹窗 | 功能① | 成员A（组长） |
| `/stock-in` | 入库管理（自动累加库存） | 功能② | 成员B |
| `/operations` | 领用/归还/废弃（自动算库存、库存不足拦截） | 功能③ | 成员C |
| `/warning` | 库存预警（低于阈值整行标红） | 功能④ | 成员D |
| `/audit` | 操作审计日志（自动记录增删改查） | 功能⑤ | 成员E |
| `/login` | 登录认证 | —— | 成员E |

## 七、常见问题

- **改了 schema.prisma 后**：执行 `npx prisma migrate dev --name 描述` 重新迁移。
- **想重置数据**：删除 `prisma/dev.db` 后重新 `npx prisma migrate dev` + `npm run seed`。
- **数据库可视化**：`npx prisma studio`（浏览器查看/编辑数据）。
